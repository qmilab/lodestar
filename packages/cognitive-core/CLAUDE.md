# @qmilab/lodestar-cognitive-core ‚Äî CLAUDE.md

This is where the epistemic chain comes alive. The core consumes typed Observations and produces Claims, EvidenceSets, Beliefs, Decisions, and Explanations.

## What lives here

- **Extractors** (`src/extractors/`) ‚Äî produce Claims from Observations. One extractor per observation schema (or the generic LLM-driven extractor for ad-hoc schemas). The opt-in `DocumentationExtractor` (`documentation.source@1`) reads *into* file content ‚Äî `package.json` deps/version, markdown title/headings/commands, exported function signatures ‚Äî for the documentation-agent proving ground; it is not a built-in, so consumers register it explicitly.

  **Generic LLM extractor (#163, ADR-0035).** `createGenericLLMExtractor(model, opts?)` returns a `ClaimExtractor` for the reserved `__generic__` fallback slot (`GENERIC_EXTRACTOR_SCHEMA_KEY`) ‚Äî what `lookupExtractor` routes to when an observation's schema has no schema-bound extractor. It is **opt-in, never a built-in** (`registerBuiltInExtractors` leaves the slot empty): replay-stable deterministic extraction stays the default, and an LLM extractor is registered deliberately. Lodestar ships **no** LLM client/key/prompt ‚Äî the consumer injects a `GenericExtractionModel` (`extractClaims(text, schema, observation) ‚Üí GenericClaimDraft[]`); the extractor stamps provenance (`source_observation_ids`, scope, sensitivity, authorship) so a draft can't lie about its origin, caps claim count + input text length, and mints each claim `extraction_method: "llm"`. **Safety:** an `llm` claim must never silently self-promote, so pair it with the `GenericAwareEvidenceLinker` (below) ‚Äî the `model_inference` downgrade is the linker's job, not the extractor's. Locked by `generic-llm-extractor-stays-unverified`.
- **World model** (`src/world-model/`) ‚Äî typed key-value store of the agent's current picture of the world. Separate from beliefs (beliefs are *about* world state; the world model captures current observed state).
- **Evidence linker** (`src/evidence-linker.ts`) ‚Äî for a new claim, find supporting and contradicting sources from prior observations and beliefs. `EvidenceLinkerLike` is the interface `CognitiveCore` depends on (so hosts can inject custom linkers); `DocAwareEvidenceLinker` (`src/doc-evidence-linker.ts`) overrides it to tag `documentation.*` observation content as `external_document` and stamp each evidence item with its source file (`independence_group: doc:<path>`), the same stance the MCP proxy takes for tool-result content. `GenericAwareEvidenceLinker` (`src/generic-evidence-linker.ts`, #163 / ADR-0035) is the partner linker for the generic LLM extractor: for an `extraction_method: "llm"` claim it stamps the source evidence `model_inference` (synthetic-trust observations stay `synthetic_probe`), so the auto-observation gate keeps the belief `unverified` even at promote-grade strength and even when a second independent LLM inference corroborates it (Parallax across LLM inferences). Non-`llm` claims fall through to the base linker. The downgrade lives here, not in the base linker, **by design**: the base linker treats `extraction_method` as non-load-bearing (the source observation's own trust sets quality ‚Äî the `evidence-linker-cross-belief-join` probe relies on `llm` claims promoting via the base linker), so opting into the generic extractor means opting into this linker too.

  **Cross-belief join (#157, ADR-0032).** Beyond the new claim's own source observations, the linker joins the claim against prior beliefs in the same scope that share its `structured_predicate.(subject, relation)` ‚Äî same `object` ‚Üí a `supports` item, different `object` ‚Üí a `contradicts` item. The `(subject, relation)` key is the **shared** exported `predicateKey` from `@qmilab/lodestar-memory-firewall` (the same one the firewall's contradiction routing uses ‚Äî single source, no drift), so the linker now takes a `ClaimStore` constructor dependency to walk `belief.claim_id ‚Üí ClaimStore.get`. **Quality inheritance:** a cross-belief item inherits the *quality* and *independence_group* of the prior belief's own strongest supporting evidence (read via the held `EvidenceStore`), so a corroborator can never lend more confidence than its own basis ‚Äî two `external_document` beliefs stay blocked by the auto-observation (Parallax) gate, while a genuinely stronger independent source clears it (a crisp `unverified ‚Üí supported` flip). The logic lives in a `protected crossBeliefItems(claim)` on the base linker that **all four** bodies call (the base plus the `Doc`/`MCP`/`Runtime`-aware subclasses, which override `linkForClaim` rather than calling `super` because `EvidenceStore.put` is a strict insert ‚Äî so the join cannot live in only one body). The linker stays **pure**: it records items on the *new* claim's evidence only; transitioning the *prior* belief to `contradicted`/`superseded` stays reflection's job (epic child B, #154). **Lifecycle + sensitivity gating:** candidate peers are pulled with `max_sensitivity: claim.sensitivity` ‚Äî the same ceiling `GatedRetrieval` applies ‚Äî so a higher-sensitivity (e.g. `secret`) belief can't lend support or contradiction to a lower-sensitivity claim and signal/launder protected state through it. The join then skips any prior belief the firewall has invalidated, isolated, hard-demoted, or holds as uncertain ‚Äî non-`clean` security (`quarantined`/`suspicious`/`malicious`), `contradicted`/`superseded` truth, `expired` freshness, a `retrieval_status` that is not `restricted`/`normal` (so `blocked`/`hidden`/`privileged_only` are out), or `confidence < 0.5` unless the belief is `user_asserted`/`policy_asserted` (the same uncertainty gate retrieval applies) ‚Äî `isEligibleJoinPeer` is an allowlist on the retrieval axis (fail-closed). Together with the `max_sensitivity` filter, the peer set now mirrors the full candidate filter `GatedRetrieval.computeAcceptedCandidates` applies. This keeps a quarantined, invalidated, or explicitly-suppressed memory from lending support through this side channel (the "quarantine is one-way" invariant holds transitively). `unverified` and `restricted` peers ARE kept (the Parallax corroboration case; `restricted` is the default adopted state). **Companion fix in `core.ts`:** a claim the join nets *negative* (it contradicts a held belief) is refused as a belief AND no longer overwrites the world model ‚Äî only positive-evidence claims update observed state, so a rejected claim can't propagate its losing value. Both locked by the `evidence-linker-cross-belief-join` probe (in-memory + Postgres).
- **Planner** (`src/planner.ts`) ‚Äî turns goals into Decisions and Action proposals, respecting ContextPolicy.
- **Reflection** (`src/reflection.ts`) ‚Äî on-demand or tail-async; rule-based in v0; produces typed `ReflectionProposal`s and applies them via `MemoryFirewall` with `by_authority: "reflection"`. Stores and the firewall are optional inputs so dry-run inspection (CLI `lodestar reflect`) works without rebuilding live state. Design contract: `docs/architecture/reflection-pass.md`. Three rules:
  - **Contradicted-belief cascade** ‚Äî a `belief.transitioned ‚Üí contradicted` event in the window flags any past `decision.made` whose `belief_dependencies` names it (`decision_dependency_flagged` ‚Üí a `Revision`).
  - **Derived supersession (#157‚Üí#154 child B).** Two `supported` beliefs in the same scope sharing a claim's `structured_predicate.(subject, relation)` but asserting different `object`s are a *derived contradiction* no sentinel flagged. Reflection proposes a `belief_supersession` (older `superseded_by` newer ‚Äî recency is the one signal it has). **Propose-only:** `run()` surfaces the proposal but never applies it, even under `apply: true` (a derived conflict is a human-adjudicated hypothesis ‚Äî world changed vs. genuine disagreement); a reviewer applies it via the public `applyProposal` (the `markSuperseded` path already existed ‚Äî only detect is new). It fires once, in the pass whose window holds the later belief's `belief.adopted` event (idempotent across passes). It reuses the evidence-linker's exported `isEligibleJoinPeer` gate + the shared `predicateKey` / `stableStringify` (single source, no drift), narrowed to `truth_status: supported`, and pairs only **equal-sensitivity** beliefs (stricter than the linker's `‚â§` ceiling ‚Äî the output names both beliefs to a human, so a cross-compartment pairing would signal a higher belief's existence in either direction). Note the interaction with the #157 ingest-time join: that join *blocks* adopting a second conflicting belief, so the derive rule's real-world targets are conflicts that entered via different paths (imports / assertions / merged sessions). Needs the belief + claim stores; a dry-run pass without them is a no-op. Authenticity is moot ‚Äî the rule reads everything substantive from the firewall-governed store and only proposes, so a forged `belief.adopted` event can at most trigger a scan of an already-legitimate belief. Locked by `reflection-derives-supersession-from-conflict`.
  - **`no_op`** ‚Äî recorded when a pass observed events but no rule fired, so the audit chain distinguishes "ran and silent" from "did not run."
- **Explanation generator** (`src/explanation.ts`) ‚Äî produces structured Explanation records for governance events.

## Invariants

1. **No claim without an observation.** Every Claim references at least one Observation. Free-form LLM extractions without a source observation are invalid.
2. **No belief without evidence.** Every Belief points to a Claim, and adoption goes through the Memory Firewall, which requires an EvidenceSet.
3. **No decision without belief dependencies.** When `ContextPolicy.require_evidence_for_decisions = true`, decisions without `belief_dependencies` are rejected before they reach the planner output.
4. **No silent retrieval.** Every retrieval into model context goes through `@qmilab/lodestar-memory-firewall`'s `GatedRetrieval`. The core does not query the belief store directly.
5. **Reflection never auto-commits.** Reflection outputs are always proposals. The Memory Firewall (with appropriate authority) is the only path to commit.
6. **Working memory is ephemeral.** Anything in `working` is gone at session end. If something needs to survive, it must be promoted to episodic or semantic memory through the firewall.

## What does not live here

- Belief storage and lifecycle transitions: see `@qmilab/lodestar-memory-firewall`.
- Tool registry and execution: see `@qmilab/lodestar-action-kernel`.
- Trust ladder and approval surfaces: see `@qmilab/lodestar-policy-kernel`.
- Probes, sentinels, and calibrators: see `@qmilab/lodestar-harness`.
